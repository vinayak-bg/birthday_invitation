üî• Firebase Setup Instructions

Follow these steps to configure Firebase for your Birthday Invitation website.

‚ö†Ô∏è Important: Firebase is required for this application to work. The free tier (Spark Plan) is sufficient for most personal use cases.

Step 1: Create a Firebase Project

  1. Go to Firebase Console
  2. Click "Add project" or "Create a project"
  3. Enter a project name (e.g., "birthday-invitation")
  4. Disable Google Analytics (optional for this project)
  5. Click "Create project" and wait for it to be ready

Step 2: Register Your Web App

  1. In your Firebase project, click the web icon (</>) to add a web app
  2. Enter an app nickname (e.g., "Birthday Invitation Website")
  3. Important: Check "Also set up Firebase Hosting"
  4. Click "Register app"
  5. Copy the Firebase configuration object (you'll need this in the next step)

Step 3: Enable Authentication

  1. In the Firebase Console, go to Build ‚Üí Authentication
  2. Click "Get started"
  3. Click on the "Sign-in method" tab
  4. Enable "Email/Password" provider
  5. Click "Save"

Step 4: Set up Firestore Database

  1. In the Firebase Console, go to Build ‚Üí Firestore Database
  2. Click "Create database"
  3. Select Start in production mode
  4. Choose a Cloud Firestore location (closest to your users)
  5. Click "Enable"

Step 5: Configure Firestore Security Rules

  1. In Firestore Database, go to the "Rules" tab
  2. Replace the existing rules with the following:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Events collection - only owner can read/write
    match /events/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    
    // Invitations collection
    match /invitations/{invitationId} {
      // Owner can read/write their invitations
      allow read, write: if request.auth != null && 
                           request.auth.uid == resource.data.userId;
      
      // Anyone with the link can read and update RSVP
      allow read, update: if true;
    }
  }
}
  • Click "Publish"

    File 1: app.js

    Replace the firebaseConfig object with your Firebase configuration:

    storageBucket: "YOUR_PROJECT_ID.appspot.com",
    messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
    appId: "YOUR_APP_ID"
};

    File 2: rsvp.js

    Update the same firebaseConfig object in this file as well.

    ‚ö†Ô∏è Note: You'll need to update the configuration in both files. In a production setup, you'd use a build tool to share this configuration, but for simplicity, we're using plain JavaScript.

    Step 7: Test Locally

    1. Open index.html in a web browser
    2. Click "Get Started" or "Admin Login"
    3. Create an admin account with any email/password
    4. Fill in event details and create an invitation link
    5. Open the RSVP link in a new tab to test the guest experience

    Step 8: Deploy to GitHub Pages

    1. Create a new GitHub repository (e.g., "birthday-invitation")
    2. Initialize Git in your project folder:
      3. git init
git add .
git commit -m "Initial commit: Birthday invitation website"
    3. Add your GitHub repository as remote:
      4. git remote add origin https://github.com/YOUR_USERNAME/birthday-invitation.git
    4. Push to GitHub:
      5. git branch -M main
git push -u origin main
    5. Go to your repository on GitHub
    6. Click Settings ‚Üí Pages
    7. Under "Source", select main branch
    8. Select / (root) folder
    9. Click "Save"
    10. Your site will be published at https://YOUR_USERNAME.github.io/birthday-invitation/

    Optional: Custom Domain

    If you want to use a custom domain instead of github.io:

    1. Add a CNAME file to your repository root with your domain name
    2. Configure DNS settings with your domain provider
    3. Follow GitHub's custom domain guide

    üîí Security Tips

    • Your Firebase API key is safe to include in client-side code
    • Security is enforced through Firestore Security Rules, not by hiding the API key
    • Keep your Firebase Admin account credentials private
    • Regularly review your Firestore usage in the Firebase Console
